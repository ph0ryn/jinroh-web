# ゲーム設計概要

この文書は、人狼ゲーム本体の設計を扱う。

`docs/spec.md` は Account、Room、Player、Realtime などの基盤仕様を扱う。
この文書では、役職、ルール、アクション、終了判定、結果判定を扱う。

## 基本方針

ゲームロジックは、役職を中心に拡張できる構造にする。

役職ごとの差分はできるだけ `Role` 側に集める。
ただし、`Role` が直接 game state を書き換える設計にはしない。

```text
Role
  役職ごとの性質、制約、反応、判定を定義する

Game Engine
  Role から effect や判定結果を集める
  順序や衝突を解決する
  game state を更新する
```

重要な分離。

- 役職ごとの知識は `Role` に寄せる
- 状態更新の責任は `Game Engine` に寄せる
- 終了/継続判定と、各 Player の結果判定は分ける
- Realtime は通知であり、ゲーム状態の正本ではない
